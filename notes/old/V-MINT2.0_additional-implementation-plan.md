# V-MINT2.0 — 追加実装計画メモ（要件定義前）

## Purpose
- DB側ゴールデンテスト完了後に着手する追加実装の、現時点要望を整理する。
- 本メモは草案であり、正式仕様はテスト完遂後に要件定義として確定する。

## 前提
- 対象プロジェクト: `V-MINT2.0`
- 現時点の優先事項は移行テスト完遂（P0 + Data/DB checks）
- 追加実装は「現行V-MINTにない機能」を対象とする

## 実装テーマ（現時点）

### 1) PIN認証の非ハードコード化（Supabase Auth活用）
- 現状:
  - PIN認証画面はハードコード寄りの実装
- 要望:
  - Supabase Auth を利用した認証方式へ変更
  - ハードコード値に依存しない運用にする
- 要件定義時の検討メモ:
  - 認証方式（メール/パスワード、Magic Link、電話番号、カスタム等）
  - 権限管理（店舗単位ロール、閲覧/入力権限）
  - セッション有効期限、ログアウト導線、端末共有時の挙動
  - 移行手順（既存PIN運用からの切替と暫定互換）

### 2) 在庫量確認モードの「ダッシュボードモード」化
- 現状:
  - 在庫量確認モードは単一画面
- 要望:
  - モード名を「ダッシュボードモード」に変更
  - 配下に複数サブモードを持つ構成へ拡張

#### サブモード1: 在庫量確認（拡張版）
- 現行の在庫量確認モードをベースに継続
- 追加要望:
  - 各拠点の在庫量を表示できること
  - 前月消費量を表示できること
  - ヘッダー等にチェックボックスを設け、
    - 各拠点在庫
    - 総在庫
    の表示/非表示を制御できること
- 要件定義時の検討メモ:
  - 既定表示（初期チェック状態）
  - 表示設定の保持（セッション内のみ / 永続化）
  - モバイル時の可読性（列数増加時のUI）

#### サブモード2: 棚卸し結果確認
- 要望:
  - 各拠点の棚卸し入力結果を一覧表示
  - 拠点 + 対象月の選択で、該当銘柄の詳細を表示
- 表示項目（要望ベース）:
  - 当月のタッパー
  - 当月の物販
  - 当月の在庫
  - 当月トータル棚卸し結果
  - 前月棚卸し結果
  - 前月消費量
  - 当月移動量
  - 当月消費量
- 要件定義時の検討メモ:
  - 指標の計算定義（どのログ/ステータスを採用するか）
  - 端数処理と表示単位（g固定か、補助単位併記か）
  - フィルタ/並び替え/CSV出力の要否

## スコープ外（現時点）
- 画面デザインの最終確定
- Authプロバイダ詳細の最終確定
- 既存データ移行バッチの最終方式

## テスト完遂後の次アクション（予定）
1. 本メモを起点に正式要件定義を作成
2. 優先度・リリース順を決定（Auth先行 or ダッシュボード先行）
3. 実装チケット化（画面/API/DB/RLS/運用）
4. 検証観点（受け入れテスト）を追加定義
