# V-MINT2.0 現場スタッフ向け 取扱説明書

> 対象読者: 事務所・馬場本店・中野店・馬場2号店の各責任者  
> 最終更新: 2026-05-11（スクリーンショット追加・PDFレイアウト最適化）

---

## はじめに

**V-MINT2.0** は、シーシャ在庫の棚卸し・補充依頼・移動記録・原価計算をスマートフォン/PCでおこなうシステムです。  
アプリのインストールは不要です。下記URLを開いてアクセスします。
https://vangvieng-app.pages.dev/

---

## 基本操作

### アプリを開く

ブラウザでアプリのURLを開くと **PIN入力画面** が表示されます。

![[screenshots/01_pin_auth.png]]

- 数字をタップするたびに上の丸が青く塗りつぶされます
- 4桁入力すると自動で認証されてポータルメニューへ進みます
- 間違えた場合は **⌫ ボタン** で1桁ずつ削除できます

> PINがわからない場合は塚本に確認してください。

---

### ポータルメニュー

認証後、以下の6つのカードが表示されます。

![[screenshots/02_portal_menu.png]]

| カード | 色 | 主な用途 |
|---|---|---|
| 棚卸し入力 | 青 | 月末の棚卸し数量を入力する |
| 原価計算 | 紫 | 月次の原価データを入力・確認する |
| 補充依頼 | オレンジ | 在庫補充が必要なフレーバーを依頼する |
| 移動記録 | 緑 | フレーバーの店舗間移動を記録する |
| ダッシュボード | 赤 | 在庫量・棚卸し結果を確認する |
| 管理者画面 | グレー | フレーバー表示設定・パッケージサイズ設定・新規登録（管理者のみ） |

## 棚卸し入力

月末に各店舗でおこなう棚卸しデータの入力です。

### 全体の流れ

```
[Step 0: 開始設定]
  店舗・対象月・実施日を選ぶ
        ↓
[Step 1: タッパー確認]
  タッパーの有無と数量を入力
        ↓
[Step 2: 在庫量入力]
  タッパー在庫量（g）を入力
        ↓
[Step 3: 物販入力]
  物販個数を入力
        ↓
[Step 4: 消費量チェック]
  警告を確認して完了
```

---

### Step 0: 開始設定

![[screenshots/03_inventory_top.png]]

1. 「棚卸し入力」カードをタップ
2. **対象店舗** をドロップダウンから選ぶ（自分の店舗）
3. **対象月**（年・月）を選ぶ（通常は自動でセットされます）
4. **実施日**（今日の日付）を確認する
5. 「入力を開始する」をタップ

> 対象月は自動で未実施の棚卸し月がセットされます。変更が必要な場合のみ手動で選択してください。

---

### Step 1: タッパー確認

フレーバーごとにタッパーの使用状況を入力します。

![[screenshots/10_inventory_step1.png]]

- **タッパーあり** → トグルをON（**青**）にして数量（g）を入力
- **タッパーなし** → トグルをOFF（**グレー**）のまま ← 重要！

> **0入力とOFFは別物です。**  
> タッパーが存在しない場合は必ずトグルをOFFにしてください。  
> 「0g入力」は「タッパーはあるが空」を意味します。

---

### Step 2〜3: 在庫量・物販入力

ブランドごとに銘柄が並んでいます。

![[screenshots/11_inventory_step2.png]]

![[screenshots/12_inventory_step3.png]]

- **在庫量（g）**: タッパーに残っている量をグラム単位で入力
- **物販個数**: 物販として保管中のパック数を入力

入力が完了したら画面下部の「**次へ**」をタップします。

---

### Step 4: 消費量チェック

前月データと今月入力値を自動比較した結果が表示されます。

![[screenshots/13_inventory_step4.png]]

| 表示色 | 意味 | 対応 |
|---|---|---|
| 🔴 赤（消費量がマイナス） | 在庫が増えている → 入力ミスの可能性大 | 数値を再確認 |
| 🟡 黄（500g超消費） | 消費量が多い → 入力ミスかどうか確認 | 数値を再確認 |
| 🔴 赤（事務所の場合） | 事務所は消費が発生しないはず | 必ず確認 |

警告を確認して問題なければ「**完了する**」をタップ。

> 警告があっても保存は可能ですが、内容を必ず確認してから保存してください。

---

### 途中保存について

```
自動保存: 入力を止めてから 30秒後 に自動保存
手動保存: 画面右上の [途中保存] ボタン → 即時保存

次回開いたとき:
┌──────────────────────────────────────┐
│  途中保存データがあります             │
│  [復元する]  [破棄する]              │
└──────────────────────────────────────┘
→「復元する」をタップして続きから再開
```

> ブラウザを閉じても途中保存データは消えません。

## 移動記録

店舗間でフレーバーを動かすときに使います。

### 移動の全体フロー

```
移動元の店舗                      移動先の店舗
      │                                 │
      │  ①「起票」で移動内容を記録      │
      │─────────────────────────────→   │
      │                                 │
      │          ②フレーバーを移動      │
      │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─→   │
      │                                 │
      │  ③「検品」で受け取りを確認      │
      │   ←─────────────────────────────│
```

### 開始設定

![[screenshots/06_transfer_top.png]]

1. 「移動記録」カードをタップ
2. **対象月**を選ぶ

> **対象月の選び方:**  
> 当月の棚卸しが全拠点完了済みで、まだ月をまたいでいない場合は「**翌月**」を選んでください。  
> 例）3月の棚卸し完了後・3/30に移動する場合 → **4月**を選択  
> 基本的にはシステムがデフォルトでセットする対象月のままで問題ありません。

3. **実施タスク**を選ぶ

| タスク | 使うタイミング | 操作する店舗 |
|---|---|---|
| 📋 起票 | 移動元が「送る内容」を記録 | **移動元** の店舗 |
| ✅ 検品 | 移動先が「受け取った」ことを確認 | **移動先** の店舗 |
| 📦 入荷 | 新規仕入れを記録 | 仕入れた店舗 |
| 🗑️ 廃棄 | 廃棄したフレーバーを記録 | 廃棄した店舗 |
| ✏️ 修正 | 入力済みの記録を後から訂正 | どこでも可 |

---

### 起票（移動元の店舗が操作）

![[screenshots/20_transfer_issue.png]]

1. 「起票」を選び、**移動元店舗**（自分の店舗）を選ぶ
2. **移動先店舗**を選ぶ
3. **実施日**を確認して「移動記録を開始する」をタップ
4. 銘柄を選んで**数量（g）**を入力
5. 「保存する」をタップ

---

### 検品

1. 該当タスクを選び、店舗・月を確認して「開始する」
2. 対象の記録を選択して内容を確認
3. 実態と違いがあれば修正
4. 「完了」をタップ

---

### 入荷・廃棄

基本的には「起票」と同じです。ただし、入荷と廃棄に**検品は不要**です。

**新しい銘柄が入荷**した場合、トップ画面から追加できます。管理者画面からもできるので、不安な場合は塚本に連絡してください。

---

### 修正

入力済みの記録を変更する場合に使います。

1. 「修正」を選んで対象月・記録を選ぶ
2. 銘柄の数量を変更、または行を削除
3. 「保存」をタップ

> 修正時は確認ポップアップが表示されます。内容をよく確認してから「はい」をタップしてください。

---

## 補充依頼

補充依頼をおこなう機能です。

![[screenshots/05_request_top.png]]

### 操作手順

1. 「補充依頼」カードをタップ
2. **依頼元店舗**（自分の店舗）を選ぶ
3. **対象月**を選ぶ（移動記録と同じ選び方）
4. 「依頼を開始する」をタップ
5. 補充が必要な銘柄に数量を入力
6. 「内容を確認」をタップ
7. 依頼文をコピーしてLINEで連絡

![[screenshots/30_request_step1.png]]

> 全拠点の補充依頼が横並びで確認できます。他店舗の状況を参考にしながら入力できます。

## ダッシュボード

在庫状況を確認するための画面です。**入力はおこないません。**

### サブモードの種類

![[screenshots/07_dashboard_top.png]]

#### 在庫量確認

現在の全拠点の在庫量を銘柄ごとに表示します。

![[screenshots/40_dashboard_overview.png]]

画面上のチェックボックスで「総在庫量」「各拠点在庫」の表示を切り替えられます。

---

#### 棚卸し結果確認

指定した月・拠点の棚卸し結果を確認できます。

| 項目 | 内容 |
|---|---|
| 当月タッパー | タッパー在庫量 |
| 当月物販 | 物販在庫量 |
| 当月在庫 | 合計在庫量 |
| 前月消費量 | 前月に消費した量 |
| 当月移動量 | 今月の移動記録分 |
| 当月消費量 | 今月の消費量（色で警告あり） |

**当月消費量の色の意味:**

```
🔴 赤（事務所）  → 0以外の値がある → 要確認
🔴 赤（その他）  → マイナスになっている → 要確認
🟡 黄（その他）  → 500g以上消費している → 要確認
⬜ 白（正常）    → 問題なし
```

---

#### 実質原価

月次の原価指標推移グラフを確認できます（原価計算データ入力後に反映）。

## 原価計算

月次の原価データを入力する機能です。  
対象: **馬場本店・中野店・馬場2号店**（事務所は除く）

### 開始設定

![[screenshots/04_cost_top.png]]

1. 「原価計算」カードをタップ
2. **対象店舗**を選ぶ
3. **対象月**を選ぶ
4. **モード**を選ぶ

| モード | 用途 |
|---|---|
| 💨 シーシャ | シーシャ本数・フレーバー・炭の消費量を入力 |
| 🥤 ドリンク | ドリンク発注記録を追加・管理する |

---

### シーシャモード

「開始する」をタップすると5ステップの入力フォームに進みます。

```
進捗バー:
[Step1] → [Step2] → [Step3] → [Step4] → [Step5]
集計期間   販売数   フレーバー    炭      プレビュー
```

#### Step 1: 集計期間

![[screenshots/50_cost_step1.png]]

- **開始日** = 前月終了日の翌日
- **終了日** = 今月末日

---

#### Step 2: シーシャ販売数

![[screenshots/51_cost_step2.png]]

> 自動計算: **提供本数** = 初回 + おかわり + スタッフ + イベント  
> 自動計算: **来客数** = チャージ + おかわり + スタッフ + イベント

---

#### Step 3: フレーバー消費量

ブランド別に物販個数を入力します。

![[screenshots/52_cost_step3.png]]

> Azure Gold/Black Line は1ブランドに集約されています。  
> Tangiers各種も1ブランドとして入力してください。

---

#### Step 4: 炭消費量

炭の種類ごとに消費量（kg）を入力します。

![[screenshots/53_cost_step4.png]]

---

#### Step 5: プレビュー・保存

![[screenshots/54_cost_step5.png]]

| 指標 | 内容 |
|---|---|
| A | 1本あたりフレーバー使用量（g） |
| B | 1本あたりフレーバー原価（¥） |
| C | 1本あたり炭使用量（g） |
| D | 1本あたり炭原価（¥） |
| E | 1人あたりドリンク代（¥） |
| F | 1人あたりシーシャ本数 |
| G | 1人あたり実質原価（¥） |

内容を確認して「**保存する**」をタップしてください。

---

### ドリンクモード

店舗・月を選ぶと自動でドリンク発注記録が読み込まれます。

![[screenshots/55_cost_drink.png]]

#### 追加の手順

1. **発注日**を選ぶ
2. **金額（¥）**を入力
3. **備考**を入力（例: 水×3、お茶×2）
4. 「追加する」をタップ

#### 編集・削除

追加済みの記録はリストに表示されます。  
各行の **✏️（編集）** または **🗑️（削除）** ボタンから操作できます。

> 月内に複数回発注した場合は、その都度「追加する」で記録を増やせます。

## 管理者メニュー（管理者のみ）

カードタップ後、管理者PINで認証が必要です。

### フレーバー表示設定

棚卸し・補充依頼・移動記録に表示するフレーバーの表示/非表示を一括管理できます。

```
┌──────────────────────────────────────┐
│  フレーバー表示設定                   │
│                                      │
│  [✅] ダブルアップル     Al Fakher   │
│  [✅] レモンミント       Al Fakher   │
│  [☐ ] 廃番フレーバー    Al Fakher   │ ← 非表示
└──────────────────────────────────────┘
```

- チェックをONにすると各画面に表示
- チェックをOFFにすると各画面から非表示

> 廃番になったフレーバーはここで非表示にしてください。

### 新フレーバー追加

新しいブランド・銘柄をマスタデータに登録します。

1. ブランド名・フレーバー名を入力
2. 各項目を設定して「登録する」をタップ

## よくある質問

**Q. 入力途中でブラウザを閉じてしまいました**

```
再度アプリを開くと表示されるポップアップ:
┌────────────────────────────────────┐
│  途中保存データがあります           │
│  [復元する]  [破棄する]            │
└────────────────────────────────────┘
```

A. 「**復元する**」をタップすると途中から再開できます。

---

**Q. 対象月を間違えて入力してしまいました**  
A. 移動記録の「**修正**」モードから対象記録を選んで訂正できます。棚卸しデータの場合は塚本に連絡してください。

---

**Q. 赤い警告が出ましたが保存できますか？**  
A. 保存は可能です。ただし警告内容を必ず確認し、入力ミスがないか再チェックしてから保存してください。

---

**Q. 「アクセスエラー」と表示されました**

```
┌──────────────────────────────┐
│  ⚠️ アクセスエラー           │
│  データの取得に失敗しました   │
│  [詳細メッセージ]            │
└──────────────────────────────┘
```

A. ネットワーク接続を確認してください。接続に問題がなければ塚本に報告してください。

---

**Q. PINを忘れました**  
A. 塚本に確認してください。

---

**Q. トグルとは何ですか？**

```
ON（青） :  ● ─────────────
           （右に寄っている）

OFF（グレー）: ─────────── ●
              （左に寄っている）
```

A. スイッチのようなボタンです。タップするたびにON/OFFが切り替わります。棚卸し入力のタッパー確認画面で使います。
