# V-MINT2.0 原価計算機能 テスト計画

## 目的
- `V-MINT2.0_reuquirements_cost_calculation` の要件を満たしているかを、ローカル環境で機能・計算・データ保存の観点から検証する。
- 原価計算モードとダッシュボード実質原価サブモードの受け入れ可否を判断できる状態にする。

Source: [[V-MINT2.0/notes/V-MINT2.0_reuquirements_cost_calculation]]

## 対象範囲
- 原価計算モードの入力/保存/上書き
- ドリンク発注の随時入力と月次集計
- 実質原価のリアルタイムプレビュー
- ダッシュボードの店舗切替/推移グラフ/サマリーテーブル
- Supabase テーブルへの保存整合（`cost_reports`, `drink_orders`, `flavor_brand_sales`）

## 対象外
- 本番データ移行作業
- パフォーマンステスト（大量同時アクセス）
- 権限/RLS のセキュリティ監査

## テスト前提
- ローカルサーバー起動済み（`npm run dev`）
- Supabase 接続設定済み（`.env` の URL/Key が有効）
- テスト対象店舗: 馬場本店 / 中野店 / 馬場2号店
- 比較用に `cost_calculation_baba2nd_2026.xlsx` を参照できる状態

## テストデータ
- 対象月: 2026年5月
- 集計期間: 2026-05-01 〜 2026-05-31
- 炭総消費量（kg）: 120
- ブランド別物販個数（抜粋）:
  - Al Fakher: 30
  - Azure Gold/Black: 12
  - Tangiers: 8
  - VV: 5
- ドリンク発注:
  - 2026-05-03 / 12500 / コーラ・ジンジャー
  - 2026-05-14 / 8600 / 紅茶・炭酸水
  - 2026-05-25 / 9400 / ジュース

## テスト観点とケース

### 1. 画面遷移・基本入力
- TC-01: Portal から「原価計算」へ遷移できる
- TC-02: 店舗選択肢が 3 店舗のみ（事務所なし）
- TC-03: 対象月、開始日、終了日を入力できる
- TC-04: 開始日が前月終了日の翌日ルールに沿って入力運用できる

### 2. 入力バリデーション
- TC-05: 数値項目に文字列を入れた場合に不正入力として扱われる
- TC-06: 炭消費量、販売個数、金額で負数を防止できる
- TC-07: 必須項目未入力時に保存できない
- TC-08: 極端に大きい値入力時も UI 崩れ・計算破綻がない

### 3. ブランド同期・入力UI
- TC-09: ブランド一覧がマスタ由来で表示される
- TC-10: Azure Gold/Black を単一項目で入力できる
- TC-11: Tangiers 系を単一項目で入力できる
- TC-12: ブランド追加時に入力欄へ反映される（動的同期）

### 4. ドリンク発注（随時入力）
- TC-13: 発注日/金額/備考を1件保存できる
- TC-14: 同月に複数件を追加できる
- TC-15: 入力済み発注が一覧表示される
- TC-16: 月次計算時に同月発注の合計金額が使われる

### 5. 計算ロジック
- TC-17: 定数（1g単価40円、炭1kg600円、料金改定値）が反映される
- TC-18: 「提供消費量」の算出が式どおり
- TC-19: 「シーシャ1本当たり原価」が式どおり
- TC-20: A〜G の算出結果が Excel 比較で許容差内（目安: 四捨五入後 ±1円）
- TC-21: 入力値変更時、期待実質原価プレビューがリアルタイム更新

### 6. 保存・再編集
- TC-22: 保存後に `cost_reports` へ月次データが作成される
- TC-23: ブランド販売が `flavor_brand_sales` に保存される
- TC-24: ドリンク発注が `drink_orders` に保存される
- TC-25: 同店舗・同対象月を再選択すると既存値が表示される
- TC-26: 上書き保存後に最新値へ更新される

### 7. ダッシュボード（実質原価サブモード）
- TC-27: 店舗切替で表示データが切り替わる
- TC-28: 月次推移グラフ（実質原価率/1本当たり原価/客単価）が表示される
- TC-29: サマリーテーブルが表示される
- TC-30: 原価計算モードで更新した値がダッシュボードへ反映される

## 実行手順（推奨順）
1. TC-01〜TC-04（画面と基本入力）
2. TC-05〜TC-12（入力制御とブランド同期）
3. TC-13〜TC-16（ドリンク随時入力）
4. TC-17〜TC-21（計算検証）
5. TC-22〜TC-26（保存・再編集）
6. TC-27〜TC-30（ダッシュボード反映）
