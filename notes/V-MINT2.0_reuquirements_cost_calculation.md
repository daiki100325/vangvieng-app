# V-MINT 原価計算機能 統合要件定義書

## 1. プロジェクト概要

在庫管理システム「V-MINT2.0」に、店舗ごとの原価計算および分析機能を統合する。スプレッドシートで運用されている複雑な原価計算ロジックを自動化し、データの蓄積と可視化をSupabaseを介して実現する。
正本として、現行のスプレッドシートをExcel形式でダウンロードした cost_calculation_baba2nd_2026.xlsx をV-MINT2.0ディレクトリ直下に格納済み。当該ブックは馬場2号店用。

## 2. 新設機能の目的

- **「原価計算」モード**: 月次のフレーバーのブランド別物販数、炭の総消費量、ドリンク発注額を入力し、実質原価を算出する。フレーバーのブランド別物販数と炭の総消費量については月末に一回のみ入力するが、ドリンク発注額については月内で複数回の発注が発生するため、発注のたびに入力できるようにする。
    
- **ダッシュボードモード配下サブモード「実質原価」**: 実質原価の推移を可視化し、経営判断の迅速化を図る。
    

## 3. 機能要件

### 3.1 「原価計算」モード (Cost Calculation Mode)

各店舗の月次データを入力・保存するインターフェース。

- **基本情報入力**:
    
    - 集計期間（開始日・終了日）の設定。
	    - 開始日については、前月分の開始日の翌日。
    - 対象月（例：2026年5月分）の指定。
    - 対象店舗の指定。
	    - 選択肢は、馬場本店、中野店、馬場２号店。事務所は含まない。
        
- **変動費・消費量入力**:
    
    - 炭の総消費量 (kg)
	    - ニャンココ FLAT（提供）
	    - ニャンココ FLAT（物販）
	    - ニャンココ CUBE（物販）
	    - KINGCO FLAT（提供）
	    - KINGCO FLAT（物販）
	    - KINGCO CUBE（物販）
		        
    - フレーバーブランド別の物販個数
	    - 対象ブランド（ブランドマスタテーブルと動的に同期させたい。以下は現行の一覧）
		    - Al Fakher
		    - Al Waha Elite Edition
		    - Al Waha JSE
		    - Afzal
		    - Azure Gold/Black
			    - 物販はAzure Gold LineとAzure Black Lineを区別せずに数量管理している
		    - Debaj
		    - Dozaj
		    - Fantasia
		    - Fumari
		    - Gixom
		    - Hookafina
		    - Mazaya
		    - Musthave
		    - Oduman Blend
		    - Pure Tobacco
		    - Revoshi
		    - Starbuzz/Bold
		    - Social Smoke
		    - Trifecta
		    - Tangiers
			    - 物販はTangiers Noir、Tangiers Birquq、Tangiers Burley、Tangiers F-Lineを区別せずに数量管理している
			- VV (Vang Vieng Mix)
        
- **ドリンク発注管理 (随時入力)**:
    
    - 発注日、金額、備考（ドリンク名など）を入力し、非同期でSupabaseへ保存。
        
    - 計算時にはその月の合計金額を自動集計して使用する。
        
- **計算項目 (スプレッドシートロジックの再現)**:
    
    - A: シーシャ1本当たりのフレーバー使用量 [g]
        
    - B: シーシャ1本当たりの原価 [¥]
        
    - C: シーシャ1本当たりの炭の使用量 [g]
        
    - D: シーシャ1本当たりの炭の原価 [¥]
        
    - E: 1人あたりのドリンク代 [¥]
        
    - F: 1人あたりのシーシャオーダー本数 [本]
        
    - G: 1人あたりの実質原価 [¥]
        

### 3.2 ダッシュボード - 実質原価サブモード

保存された計算結果を閲覧するモード。

- **店舗切り替え**: 複数店舗のデータを切り替えて表示。
    
- **月次推移グラフ**: 実質原価率、1本当たり原価、客単価の推移をチャート化。
    
- **サマリーテーブル**: スプレッドシートの「集計欄」の内容を一覧で確認。

## 5. 計算ロジック詳細

現行版Excel内の以下の定数と計算式をフロントエンドまたはデータベースのViewで実装する。

- **定数**:
    
    - フレーバー1g単価: ¥40
        
    - 炭1kg単価: ¥600
        
    - チャージ料: ¥900（2026円5月から改定）
    - シーシャ１本目の値段: ¥1900
    - シーシャおかわりの値段: ¥1800（2026年5月から改定）
    - スタッフシーシャの価格: ¥1800（2026年5月から改定）
        
- **計算式例**:
    
    - `提供消費量 = 総消費量 - (販売個数 * 1パケ当たりグラム数)`
        
    - `シーシャ1本当たり原価 = (フレーバー使用量 * 1g単価) + (炭使用量 * 1kg単価 / 1000)`
        

## 6. UI/UX デザイン方針

- **入力フォーム**:
    
    - フレーバーブランド一覧は、既存の在庫マスタから動的に生成する。
        
    - 数値を変更するたびに、画面下部の「期待される実質原価」がリアルタイムに更新されるプレビュー機能を備える。
    - 入力ミスの修正に備え、棚卸し入力と同様、データ送信後でも同じ店舗・同じ対象月をトップ画面で選択すれば、送信済みの数値の閲覧と上書き入力ができるようにしたい。
    - 
        
- **ドリンク入力**:
    
    - ドリンクの発注は月内に複数回発生するため、随時入力ができ、かつその対象月でそれまでに入力済みのものが移動記録モードのように一覧で確認できると良い。
