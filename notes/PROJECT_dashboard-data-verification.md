# PROJECT — Dashboard Data Verification

## Purpose
- 明日移行時に、`V-MINT2.0` のダッシュボードモードで表示される数値が、正本データと一致しているかを確認する。
- UI操作の成否ではなく、`在庫量確認` / `棚卸し結果確認` の数値整合を重点確認する。
- Source: [[PROJECT/notes/PROJECT_golden-test-checklist]]

## Scope
- 対象画面:
  - `ダッシュボードモード > 在庫量`
  - `ダッシュボードモード > 棚卸し結果`
- 対象データ:
  - 正本CSV / 正本集計値
  - Supabase に投入済みの `brands`, `flavors`, `inventory_logs`, `transfer_logs`
- 対象月:
  - 主要確認: 直近移行対象月
  - 追加確認: 前月、必要に応じて年跨ぎ月

## Pre-check
- [x] `.env.local` が正しい Supabase 接続先を向いている ✅ 2026-04-30
- [x] 対象月の normalized CSV 取込が完了している ✅ 2026-04-30
- [x] `brands` / `flavors` / `inventory_logs` / `transfer_logs` の投入漏れがない ✅ 2026-04-30
- [x] 比較元の正本データ（CSVまたは既存運用側集計）が手元で参照できる ✅ 2026-04-30
- [x] 物理キーボードで PIN 解除でき、ダッシュボードモードを開ける ✅ 2026-04-30

## Expected Logic

### 1. 在庫量
- `総在庫量`:
  - 全拠点在庫の合計
- `各拠点在庫`:
  - `office`, `baba_main`, `nakano`, `baba_2nd` ごとの在庫
- `前月消費量`:
  - 前月について `前々月棚卸し + 前月completed移動 - 前月棚卸し`

### 2. 棚卸し結果
- `当月タッパー1`
- `当月タッパー2`
- `当月物販`
- `当月在庫`
- `当月トータル棚卸し結果`
- `前月棚卸し結果`
- `前月消費量`
- `当月移動量`
- `当月消費量`
- `当月消費量` 強調（`office` 拠点: 0 以外は赤 / その他拠点: マイナスは赤、500g 以上は黄）

## Test Procedure

### A. 在庫量確認 ✅ 2026-04-30（正本と整合）
1. `ダッシュボードモード` を開く
2. `在庫量` を選択する
3. 対象月を選択する
4. 一覧表示後、以下を確認する
   - [x] 行数がおおむね期待どおりで、主要ブランド/フレーバーが欠落していない
   - [x] `総在庫量` が正本集計値と一致する
   - [x] `各拠点在庫` の4列が正本集計値と一致する
   - [x] `前月消費量` が正本集計値と一致する
   - [x] `総在庫量 <= 0` の行だけが強調表示される
5. 表示切替を確認する
   - [x] `総在庫量` の表示ON/OFFが正しく効く
   - [x] `各拠点在庫` の表示ON/OFFが正しく効く
   - [x] 表示切替後も数値自体は変わらない
6. ブランドフィルタを確認する
   - [x] ブランド絞り込みで対象行だけが表示される
   - [x] フィルタ解除で全件へ戻る

### B. 棚卸し結果確認 ✅ 2026-04-30（正本と整合）
1. `ダッシュボードモード` を開く
2. `棚卸し結果` を選択する
3. 対象月を選択する
4. 対象店舗を選択する
5. 一覧表示後、主要サンプル銘柄で以下を確認する
   - [x] `当月タッパー1` が正本値と一致する
   - [x] `当月タッパー2` が正本値と一致する
   - [x] `当月物販` が正本値と一致する
   - [x] `当月在庫` が正本値と一致する
   - [x] `当月トータル棚卸し結果` が各内訳合計と一致する
   - [x] `前月棚卸し結果` が正本値と一致する
   - [x] `前月消費量` が正本値と一致する
   - [x] `当月移動量` が正本値と一致する
   - [x] `当月消費量` が正本値と一致する
6. `当月消費量` 列の強調表示（拠点別）を確認する
   - [x] **事務所** 選択時: `当月消費量` が **0 以外** の行が赤く強調される
   - [x] **事務所以外** 選択時: `当月消費量` が **マイナス** の行が赤く強調される
   - [x] **事務所以外** 選択時: `当月消費量` が **500 以上** の行が黄く強調される

## Sampling Guidance
- 全件目視確認が難しい場合は、最低でも以下を確認する（棚卸し結果 PASS 時はサンプル抽出で実施）
  - [x] 主要ブランドをまたいだ 5〜10 銘柄
  - [x] 在庫0またはマイナス境界の銘柄
  - [x] 前月消費量が大きい銘柄
  - [x] 当月移動量がプラス/マイナスで発生している銘柄
  - [x] 店舗差が大きい銘柄

## Notes
- 数値不一致が出た場合は、UIだけでなく `period_key`、`status=completed` 条件、店舗キー変換（`baba` -> `baba_main`）も確認する。
- 正本と比較するときは、同じ月の定義になっているか（`period_key` ベースか）を先に合わせる。
- 2026-04-30: 照合対象月と異なる `period_key`（例: 5月ラベル）で `transfer_logs` にテスト移動を入れると、在庫推定・前月消費の両方がずれる。削除・修正後に正本と整合することを確認済み。Source: [[PROJECT/TROUBLESHOOTING]]
