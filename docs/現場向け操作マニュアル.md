# 店舗業務ポータル　現場向け操作マニュアル

**対象:** スマートフォン・タブレット・PC ブラウザから利用する現場スタッフ向け  
**版:** アプリ最新版に準拠（コンポーネント分割後の画面構成）

---

## 本マニュアルについて

- 文中の **図** は、`docs/images/` フォルダ内の画像ファイルを指します。
- 画像がまだない場合は、[images/README.md](./images/README.md) の一覧に沿ってスクリーンショットを追加してください。

## 1. はじめに

### 1.1 アプリを開く

1. 担当者から案内された **URL** をブラウザで開きます。
2. 最初に **暗証番号（4桁）** の入力画面が表示されます。
3. テンキーで番号を入力すると、認証が完了し **業務選択（ポータル）** に進みます。

**図1-1**　PIN 入力画面  
![PIN入力](./images/01-pin.png)

> **注意:** PIN を連続で誤るとロックされる場合があります。分からないときは管理者に確認してください。

### 1.2 ポータル（業務の選択）

認証後、次の **4つの業務** からひとつ選びます。

| 名称 | 用途のひとこと |
|------|----------------|
| **棚卸し入力** | 月末棚卸しの数量を入力し、スプレッドシートに反映する |
| **補充依頼** | 在庫が足りないフレーバーについて、他店からの補充を依頼する |
| **移動記録** | 店舗間の移動・入荷・廃棄などを起票・検品・記録する |
| **在庫量確認** | フレーバー別の総在庫・前月消費を一覧で見る（参照のみ） |

**図1-2**　ポータル画面  
![ポータル](./images/02-portal.png)

### 1.3 どの画面からでも戻る

- 画面上部左の **←（戻る）** で、**ポータル** に戻れます。
- 入力途中の場合は **「入力途中のデータがある場合、破棄されます」** などの確認が出ることがあります。内容を確認してから **OK** を押してください。

### 1.4 ブランド（銘柄）で絞り込む

棚卸し・補充依頼・移動記録・在庫量確認では、画面右下などに **ブランド絞り込み** 用の操作があります（フレーバーが多いときに便利です）。

## 2. 棚卸し入力

### 2.1 このモードですること

各店舗の **月末棚卸し** の結果（タッパー・物販・在庫）を入力し、**送信**するとスプレッドシートに書き込まれます。

### 2.2 操作の流れ（通常店舗）

1. **棚卸しを開始** で **対象店舗・対象月・実施日** を選びます。
2. **入力を開始する** → **店舗の確認** で内容を確認し **開始する**。
3. 保存済みデータがある場合は **「保存されたデータが見つかりました。復元しますか？」** が出ます。続きから入力するなら **OK**、最初からやり直すなら **キャンセル** 後に読み込みが続きます。
4. **Step1 タッパー**（単位: g）→ **次へ**
5. **Step2 物販**（単位: 個）→ **次へ**
6. **Step3 在庫**（単位: 個）→ **次へ**
7. **Step4 入力内容確認** で内容を確認 → **完了する**
8. **「データを送信します。よろしいですか？」** で **OK** → 保存完了のメッセージ  
   - 条件により **前月消費量がマイナス** になるフレーバーがある場合は、**警告メッセージ** が追加で表示されます。内容を確認してください。

### 2.3 事務所を選んだ場合

- **タッパー・物販の入力画面は出ません。** 在庫入力から始まり、ステップ数が少なくなります（ヘッダーの Step 表示も 2 ステップ相当になります）。
- 手順の最後も **完了する** → 確認ダイアログ → 送信、まで同じです。

### 2.4 一時保存（入力の途中でポータルに戻る）

- **次へ** を押したタイミングや、**モードを抜ける直前** に、端末内（ブラウザの localStorage）へ下書きが保存されます。
- 再度同じ **店舗・月** で開始すると、**復元の確認** が表示されることがあります。

### 2.5 よくあること

- **数値は半角**で入力してください（空欄は未入力扱いです）。
- 送信後は店舗・月などがリセットされ、ポータルに近い状態に戻ります。

## 3. 補充依頼

### 3.1 このモードですること

**依頼元店舗（自分の店舗）** と **対象月** を選び、一覧から **不足分の数量（g）** を入力して依頼します。他店舗の在庫状況を踏まえた依頼作成に使います。

### 3.2 操作の流れ

1. **依頼元店舗** と **対象月** を選び **依頼を開始する**。
2. フレーバー一覧が表示されます。**在庫ゼロ非表示** のスイッチで表示を絞り込めます。
3. 必要な行に **依頼数量** を入力します。
4. **内容を確認** で確認画面へ進み、問題なければ送信手順に進みます（画面の案内に従ってください）。

**図3-1**　補充依頼の開始  
![補充依頼開始](./images/09-request-start.png)

**図3-2**　依頼ボード（一覧・入力）  
![補充依頼ボード](./images/10-request-board.png)

### 3.3 対象月について

画面上の説明に従い、**棚卸しが全拠点完了した月** と **まだ月をまたいでいないか** を意識して選んでください（例: 3月棚卸し完了後、3/30 の依頼で 4月を選ぶ、など）。

## 4. 移動記録

### 4.1 このモードですること

店舗間の **商品移動** を **起票** し、受け取り側で **検品** したり、**入荷・廃棄** などの記録を行います。業務ごとに画面が分かれます。

### 4.2 最初の画面ですること

1. **対象月** を選びます（棚卸し・月跨ぎの扱いは画面の注釈を参照）。
2. **実施タスク** を選びます。
   - **起票** … 移動元・移動先・実施日などを入力して数量を入力し、記録する流れ
   - **検品** … 届いた側の店舗で、未検品の記録を選んで内容を確認する流れ
   - **入荷** … 事務所向けの入荷記録（画面の案内に従う）
   - **廃棄** … 廃棄記録（店舗・実施日など）

**図4-1**　移動記録トップ  
![移動記録トップ](./images/11-transfer-top.png)

**図4-2**　起票または確認・送信の一例  
![移動記録起票](./images/12-transfer-issue.png)

### 4.3 操作時の注意

- **起票・検品** では、画面上部やフッターの **戻る／次へ／内容を確認／データを記録／検品を完了** などに従って進めます。
- **ブランド絞り込み** で対象フレーバーを絞ると入力しやすくなります。

## 5. 在庫量確認

### 5.1 このモードですること

**参照のみ**です。**対象月** を選ぶと、フレーバーごとに **総在庫量（g）** と **前月消費量（g）** が一覧表示されます。発注や在庫の検討の材料に使います。

### 5.2 操作の流れ

1. ポータルから **在庫量確認** を開く。
2. **対象月** を選ぶと、自動的にデータが読み込まれます。
3. 一覧をスクロールして確認。**総在庫が 0 以下** の行は強調表示されることがあります。

**図5-1**　対象月と一覧  
![在庫量確認](./images/13-stock-month.png)

### 5.3 注意

- このモードでは **数値の送信・編集はしません**（閲覧専用）。

## 6. 困ったとき

| 現象 | 試すこと |
|------|----------|
| ずっと読み込みのまま | 通信環境を確認し、しばらく待ってから再度開く。改善しない場合は管理者へ。 |
| エラーメッセージが出る | 表示された文面をメモし、**店舗・月・操作手順** と合わせて管理者へ連絡。 |
| 送信したが反映されない | 別端末でスプレッドシートを確認。キャッシュのため時間差がある場合は待つ。 |
| PIN がわからない | 管理者のみが設定・再発行できます。 |
