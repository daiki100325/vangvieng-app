## 1. プロジェクト概要

### 1.1 背景・目的

店舗スタッフが **棚卸し・補充依頼・移動記録・在庫確認** をモバイル中心に実施できるよう、既存の単一 `App.vue` 構成から **保守性・拡張性の高い構成**へ移行し、**本番運用で発生した不具合を解消**する。

### 1.2 スコープ

| 含む | 含まない（参考） |

| Vue 3 フロントのリファクタリング・不具合修正 | GAS / スプレッドシートの内部ロジック変更 |

## 2. 実施内容サマリー

### 2.1 アーキテクチャ変更（コンポーネント分割）

`App.vue` に集約されていた UI・ロジックを、以下の単位へ分割した。

| 区分 | コンポーネント |
|------|----------------|
| 業務アプリ | `InventoryApp`, `RequestApp`, `TransferApp`, `StockApp` |
| 共通 UI | `AppHeader`, `AppFooter`, `PinAuth`, `LoadingOverlay`, `BrandFilterSheet` |
| ナビゲーション | `PortalMenu` |

**効果:** 責務の分離、レビュー容易化、モード別の改修リスク低減。

### 2.2 解決した主要不具合（棚卸し）

| 課題 | 原因 | 対応 |
|------|------|------|
| 「完了する」で確認・送信が動かない | イベント／参照の取り回し不整合 | フッター emit → 親 → `ref` 経由で `submitInventory` 呼び出しに整理 |
| シート未定義・`getRange(null)` エラー | 誤 API（`getInventoryData`）＋ペイロード形式不一致 | **`getSheetData` + `sheetName` / `date` / `rowIndex` 整合** |
| フレーバー名非表示 | API キー差異 | `flavorName \|\| flavor \|\| name` フォールバック |
| 事務所でタッパー画面になる | 初期ステップ固定 | 事務所は Step 3（在庫）から開始 |
| ドラフト復元でクラッシュ | 親 `saveLocally` が `items` 未定義を保存 | **`InventoryApp` に保存処理を移管**、無効 JSON は削除 |
| 一時保存が効かない | 初期ロード直後に空データ保存／離脱時未保存 | 初期自動保存をやめ、**`beforeUnmount` + `nextStep` で保存** |
| マイナス消費の警告が出ない | 成功時分岐未実装 | `negativeConsumptionItems` を確認してアラート |
| 最下行とフッター重複 | 下余白不足 | **`padding-bottom: calc(7.5rem + safe-area)`** |

## 5. 残課題・推奨改善（非ブロッキング）

1. **`App.vue` レガシー整理**  
   棚卸し開始まわりの重複（`startInventory` / 未使用モーダル用データ）を整理し、Vue warn をゼロに近づける。
2. **店舗キーの統一**  
   棚卸し `baba` と補充依頼 `baba_main` の命名を、GAS と合わせて一本化するか、マッピング層を明示する。
3. **`AppHeader` props の完全配線**  
   `requestCurrentStoreName` 等を `RequestApp` から供給するか、未使用なら削除。
4. **E2E / 回帰テスト**  
   Playwright 等で主要フロー（棚卸し送信・ドラフト・移動記録）を自動化。
5. **GAS 側仕様書**  
   アクション名・シート名規則・列マッピングをスプレッドシート管理者と共有。
